- **GraphQL** используется там, где это реально удобно для загрузки данных, auth и части CRUD;
- **LiveRail** используется там, где действие по природе **live-first** или должно сразу синхронизироваться между клиентами;
- если действие идет через **GraphQL**, то в задаче явно указано, что после изменения состояния сервер должен отправить **realtime-уведомление через LiveRail**;
- если действие идет через **LiveRail**, то это сразу считается и transport для действия, и механизмом live-синхронизации;
- **MediaSoup** остается отдельным media-слоем;
- **DobruniaUI Vue** используется для UI;
- **MySQL + Prisma + Docker** — для хранения данных.
